Кодгайд

HTML

  • Общие правила прописаны в editorconfig. Для соблюдения единообразия и правил необходимо установить плагин в свой редактор.

  • У одиночных тегов необязательные закрывающие теги не ставятся

    Плохо: <br /> <img />

    Хорошо: <br> <img>

  • Разметка должна быть семантичной

  • Интерактивные элементы размечаются <a> или <button>.

    Плохо: <span>Нажмите сюда</span>

    Хорошо: <a>Нажмите сюда</a> <button>Нажмите сюда</button>

Картинки

  • У всех картинок явно задаются ширина и размер

  • Все иконки должны быть в svg

  • Контентные svg должны быть заинлайнены в html через спрайт

  • У заинлайненых svg должны быть прописаны ширина и высота

  • Неконтентные svg задаются фоном через css, либо контентом через псевдоэлемент.

  • Как отличить контентные svg от неконтентных: контентные меняются при взаимодействии: меняется цвет, размер и т.д. Если меняется только прозрачность, смотреть по ситуации.

CSS

  • Сетки и микросетки размечаются флексбоксами

  • Именование классов происходит по методологии БЭМ в международной нотации:

    .block .block__element .block--modifier .block__element--modifier

  • Используется ccs-препроцессор Sass

  • Для соблюдения правил и единообразия кода в проекте присутствует файл .stylelintrc.

  • Вложенности селекторов не больше двух. Псевдоэлементы, псевдоклассы, а также такие селектора, как .class__name + .class__name-two вложенности не добавляют

    Плохо: .class ul li a {}

    Хорошо: .class a {}

  • При работе с файлами препроцессоров разрешается вложенность модификаторов блока, но не элементов

    Плохо: .block { &__element { ... } }

    Хорошо: .block { &--modifier { ... } } .block__element { &--modifier { ... } }

  • Через id стилизация запрещена

  • Запрещено использовать !important для того, чтобы "перебить" более старые классы.

  • Стилизация элементов происходит по имени класса, а не по имени тега.

  • Глобальные стили запрещены.

    Плохо: ul { list-style: none; }

    Исключения: a { text-decoration: none; } img { max-width: 100%; height: auto; } body {}

  • Цвет задается шестнадцатеричным значением без сокращений. При необходимости прозрачности использовать rgba().

  • Транслит при наименовании классов не используется

  • Имена классов должны явно говорить, к чему они принадлежат

  • С осторожностью использовать имена классов для цветов, чтобы в будущем не оказаться в подобной ситуации:

    Плохо: .red-btn { background-color: green; }

  • Не использовать сокращенный синтаксис свойств

    Плохо: background: green;

    Плохо: background-color: green;

JS

  • В случае необходимости отключить работу элемента по умолчанию, необходимо использовать preventDefault.

  • Используются одинарные кавычки. В случае вложенности разрешенно комбинировать олдинарные кавычки с двойными.

  • Для именования переменных и id для js используется стиль camelCase.

  • Классам, которые прописываются только для js, необходимо добавлять префикс js-

  • Для классов, которые срабатывают в случае отключенного js, необходимо приписывать префикс no-js-

  • Имена переменных должны явно говорить о том, для чего они.

Шрифты

  • Иконочные шрифты использовтать запрещено. Если возможно, делаем элемент на css, если нет, необходимо реализовать иконку в svg.

Доступность

  • Всему тексту всегда задается контрастный фон для удобочитаемости.

    В случае фона-картинки необходимо задавать страховочный фон.

  • Все интерактивные элементы должны быть доступны с клавиатуры и прокликиваться табом.

  • Все состояния вешаются на интерактивный элемент, а не на его обертку.

    Плохо: li:hover { background: green; } li:hover a { background: green; }

    Хорошо: li a:hover { background: green; }

  • Если в стайлгайде не прописано состояние для :focus, дублируем в него hover.

    Хорошо: a:hover, a:focus {...}

  • Все элементы, которые не имеют текстового описания, необходимо подстраховывать. Например, с помощью специального класса visually-hidden.

    Хорошо: <a href="index.html"><svg>иконка вк</svg><svg class="visually-hidden">Мы Вконтакте</span></a>

  • У всех картинок прописан alt, который явно описывает, что изображено на картинке. Если на картинке изображена карта - пищем конечный адрес.

Общие правила

  • Адаптивная верстка происходит по методу mobile first.

  • Используется прогрессивное улучшение. Это значит, что при неработающем js все элементы по-прежнему работают, и пользователь может получить всю необходимую ему информацию даже при медленном интернете.

  • Верстка должна проходить тест переполнения контентом.
